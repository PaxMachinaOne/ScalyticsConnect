# Maintainers

This file defines maintainer responsibilities and review expectations for this repository.

## Maintainer Responsibilities

- Keep contribution flow healthy (issues, PRs, reviews).
- Enforce project standards and policies.
- Prioritize security and stability.
- Curate release notes/changelog updates.
- Mentor new contributors when feasible.

## Review Expectations

- At least one maintainer approval is required for non-trivial changes.
- Security-sensitive changes should receive two maintainer reviews when possible.
- Breaking changes require explicit migration notes in the PR.

## Decision Process

- Prefer consensus among active maintainers.
- If consensus is blocked, the designated lead maintainer decides.
- Decisions should be recorded in PR discussion for auditability.

## Becoming a Maintainer

Candidates are evaluated on:

- sustained, high-quality contributions
- review quality and responsiveness
- alignment with project standards and code of conduct

Maintainer changes are proposed via PR updating this file.
